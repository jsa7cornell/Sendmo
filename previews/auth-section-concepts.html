Design notes Option A — Minimal friction. Removes the "Your email" heading entirely. Google button is the obvious first choice; the "or use your email" divider makes the email path feel secondary without hiding it. After sign-in, replaces the entire auth card with a compact "identity confirmed" pill. Includes an auto-advance countdown (2 seconds) since both address and identity are now confirmed — matches the pattern used by Airbnb checkout and Stripe Link.

Option B — More explicit account framing. "Create your free account" headline sets expectations. The two-card selector (Google / Email) gives the choice visual weight without feeling like a form. After sign-in, collapses to a compact verified row with a "Use a different email?" escape hatch. The Continue button is immediately available — no countdown — trusting the user to decide when to move on.

Auto-advance logic (both options): Only fires when the address is already verified in sessionStorage AND the user just returned from OAuth. If the address is empty (new visit), stays on the page so the user can fill it in. The countdown gives a moment to confirm before advancing.
