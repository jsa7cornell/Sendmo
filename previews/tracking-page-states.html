Cross-family IA rules

  • One identity anchor — the SendMo public_code is always the bold/blue one. Carrier tracking number is demoted (Family 2 only) and never shown when it's dead (Family 3) or unscanned (Family 1).
  • One hero per state — replaces today's "celebration banner + status card + tracking history" stack-of-three with a single state-tinted hero that owns "what's happening."
  • Money is part of the story — every family surfaces "Paid: $X.XX". Family 2 carries the carrier-adjustment slot. Family 3 ties refund to the same number.
  • The page never lists the same fact twice — Today: status appears in banner + status-card + progress bar. New: each fact has exactly one home.
  • "What to do" lives on Family 1 only — print/tape/drop-off instruction is invisible in 2 and 3 because the work is done.
  • Cancel/Change row only renders when actually offered (Family 1 + viewer is authorized). Today the section exists but mostly empty for unauthorized viewers — drop the empty container.
